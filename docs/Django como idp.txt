ARQUITECTURA

Django OAuth Toolkit es una biblioteca de software de código abierto para el framework web Django. Proporciona una 
implementación completa del protocolo OAuth2, permitiendo a las aplicaciones basadas en Django actuar como proveedores 
de autorización y emitir tokens de acceso para clientes externos. Es ampliamente utilizada para asegurar APIs y gestionar 
autenticaciones entre servicios.

https://django-oauth-toolkit.readthedocs.io/en/latest/

Te permite implementar OAuth2 server.

Instalación:

pip install django-oauth-toolkit

Configuración básica:

INSTALLED_APPS = [
    "oauth2_provider",
]



D1 actúa como Identity Provider (IdP):

Gestiona usuarios
Gestiona roles
Emite tokens OAuth2

D2 y D3 actúan como Service Providers (SP):

Redirigen a D1 para login
Validan tokens
Autorizan según roles

AÑADIR ROLES AL TOKEN

accounts/oauth.py:

from oauth2_provider.models import AccessToken
from .models import UserRole

def get_roles(user):
return list(
UserRole.objects.filter(user=user)
.values_list("role__name", flat=True)
)

SIGNAL PARA INCLUIR ROLES EN EL TOKEN

from oauth2_provider.signals import token_generated
from django.dispatch import receiver

@receiver(token_generated)
def add_roles_to_token(sender, request, token, **kwargs):
user = token.user
roles = list(user.userrole_set.values_list("role__name", flat=True))
token.extra_data = {"roles": roles}
token.save()

MIDDLEWARE SSO

auth/middleware.py:

import requests

class SSOAuthMiddleware:
def init(self, get_response):
self.get_response = get_response

def __call__(self, request):

    token = request.COOKIES.get("access_token")

    if token:
        resp = requests.get(
            "http://d1:8000/o/introspect/",
            data={"token": token},
            auth=("d2-client", "secret"),
        )

        if resp.status_code == 200:
            data = resp.json()
            request.user_roles = data.get("roles", [])
            request.is_authenticated_sso = True
        else:
            request.is_authenticated_sso = False
    else:
        request.is_authenticated_sso = False

    return self.get_response(request)

FLUJO

Usuario entra en D2 o D3
Se redirige a D1
Hace login en D1
D1 emite token OAuth2 con roles
D2/D3 validan token
Acceso permitido según roles

NOTAS

D1 es la única fuente de verdad
D2 y D3 no gestionan usuarios
Los roles viajan dentro del token
El acceso se controla localmente en cada app
